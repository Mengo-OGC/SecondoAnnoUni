	Un file descriptor (FD) è un'astrazione multi linguaggio dell'os per permettere l'accesso ai file hai processi/shell.

	L'OS ha la \textbf{tabella di sistema (TS)} contenente i FD dei file attualmente aperti.
	Ogni processo ha una \textbf{file descriptor table (FDT)} di record indiretti, puntano ai record della TS, dei FD aperti dal process.

	Un processo figlio eredità una copia modificabile della FDT. Padre e figlio devono sincronizzarsi sull'accesso ai file (message passing). Rindizionamenti:
	\begin{itemize}[nosep]
		\item \textbf{Processi figlio}: Padre può cambiare le associazioni fd/stream passate al figlio.
		\item \textbf{Auto-ridirezionamento}: fd viene associato a stream diverso.
	\end{itemize}
	Processi diversi possono avere FD diversi ma con stesso valore e che referenziano file diversi.

	FD standard in POSIX: stdin=0, stdout=1, stderr=2.
	In C per scrivere si usa \texttt{<fprintf(stdout/err, "..." [, parametri])>} e per leggere si usa \texttt{<fgets(buffer, size, stdin)>}, per il fine file usare \texttt{<feof(stdin)>}.
